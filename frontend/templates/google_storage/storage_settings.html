Setup: student OAuth (Google Cloud)

Client ID and secret come from Google Cloud Console — not from PowerHUB. Full guide: ⓘ help on this page or docs/GOOGLE_DRIVE_SETUP.md.

  1. New GCP project → enable Google Drive API (APIs & Services → Library).
  2. OAuth consent screen → External → app name + emails. If Testing, add each student email under Test users.
  3. Data access scopes → add Google Drive API → drive.file (required for uploads).
  4. Credentials → Create → OAuth client ID → Web application → redirect URI: {{ suggested_redirect_uri }}
  5. Copy Client ID and Client secret (secret shown once) into the form above.
  6. Enable student OAuth uploads (separate from staff Shared drive) → Save → Validate OAuth config.

Staff checkbox alone does not enable student Profile connect. Leave SITE_URL in .env aligned with this host.

Setup: Google Workspace + staff Shared drive

For teachers/admins — org Shared drive via service account (separate from student OAuth). Details: ⓘ help → sections Google Workspace, service account, PowerHUB staff.

A. Google Workspace

  1. Shared drives → New (e.g. PowerHUB Files).
  2. Add teachers as Contributor (Google Group recommended).
  3. Copy Shared drive ID from URL: drive.google.com/drive/folders/ID
  4. Allow anyone with the link sharing in org policy (students open files without drive access).

B. Google Cloud — service account

  1. Same GCP project as student OAuth is fine → enable Google Drive API.
  2. IAM → Service accounts → Create → Keys → Add key → JSON (download once).
  3. On the Shared drive: Manage members → add client_email from JSON as Content manager.

C. PowerHUB (form above)

  1. Enable staff uploads → paste Shared drive ID + service account JSON.
  2. Save → Test Shared drive connection → Ensure PowerHUB root folder.

Workspace students: set Workspace email domain below; OAuth consent Internal in GCP. Staff do not use Profile → Connect Google.
